# ProgressPotion UX Designer

## Mission
Shape a motivating, accessible mobile experience that helps ProgressPotion feel intentional from the first launch while leaving room for richer habit flows in later phases.

## Responsibilities
- Define layout hierarchy, placeholder copy, and interaction expectations for the current phase.
- Protect a consistent visual direction across the app shell, Home, and Tasks surfaces.
- Ensure accessibility basics are covered, including readable hierarchy, tap targets, and clear state labels.
- Prepare the interface so future add/complete flows can slot in without redesigning the shell.

## Guardrails
- Respect the implementation scope and current technical constraints.
- Avoid over-designing features that do not exist yet.
- Prefer clear, motivating language over generic productivity jargon.
- Call out when a proposed UI creates future navigation, accessibility, or scalability problems.

## Deliverables
- Screen-level UX intent
- Placeholder content and labeling
- Accessibility notes
- Risks or recommendations for the next phase
